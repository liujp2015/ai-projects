返利折扣网 类似淘宝客
技术
后台管理系统使用 umijs
1 商品管理 商品是可以从购物网站的营销推广中获取 也可以手动创建 最重要的字段是 link 这个字段点去购买跳转到真实的购物网站
商品管理列表 查询/添加/修改/删除 推送功能（调用 nestjs 的 api 接口添加到 redis 的队列，消费队列向 websocket 在线用户推送商品信息）
商品的字段 id 使用 uuid 标题、描述、原价、当前折扣价、跳转的 link、是否隐藏、存储 id、品牌 id、分类 id、购物平台来源、关键字、自定义顺序（首页展示列表排序的）、是否 top（首页展示列表排序的）、创建时间、结束时间、更新时间
2 品牌管理 添加商品可以选择品牌
品牌管理列表 查询/添加/修改/删除
3 商品分类管理 给商品进行分类 在新建商品的时候可以选择分类
分类管理列表 查询/添加/修改/删除
4 权限管理
对所有的模块进行 查询/添加/修改/删除
5 角色管理
角色管理 对角色的 查询/添加/修改/删除 权限分配给角色
6 用户管理
用户管理列表 查询/修改/删除 对用户修改角色 选择角色 在 PC 前端注册的用户默认没有角色
默认一个 admin 管理员有所有的权限
7 存储管理
存储管理是对第三方的云存储配置进行管理 包括地址和 apikey 和 securitykey 等信息
8 订单管理
订单管理是从购物平台同步过来的订单
能对订单进行查询和下载
9 banner 图管理
能对网站前端的 banner 图区域进行配置 轮播图进行管理
10 网站配置
设置包括注册赠送多少积分 订单成功后赠送多少积分 所以用户表需要记录积分的总积分 剩余积分的字段
需要一个积分表 记录用户的积分增加和减少的情况
前台 PC 端使用 nextjs tailwind typescript
PC 页包含 1.登录注册 登录 Remember me 一周 忘记密码 邮箱验证码注册 2.首页 搜索框 搜索商品或者分类或者品牌,也可以通过选择框条件查询
3.banner 图展示，这里展示后台设置的 banner 图 4.商品分页列表 5.点击一个商品 展示详情 详情里面还有个推荐的其他的商品列表 自动滚动 像轮播图一样
6.nextjs 需要使用 websocket 与后端的 nestjs 连接
用户登录后把 token 存在 user_access_tokens 通过 user_refresh_tokens 确定在线的用户 向在线的用户发送要推送的商品
用户进入首页登录后需要使用 websocket 连接到 nestjs 的 websocket 的且发送 token 7.个人中心 用户可以修改自己的昵称和头像 查看自己的订单（分页） 这个订单通过 clickId 和购物平台的订单号进行关联
个人中心显示自己的积分和余额 头像和昵称等信息。

后端接口使用 nestjs
需要 redis 的订阅模式订阅商品推送列表
使用使用 websocket 与 nextjs 在线的用户连接
数据库用 pgsql
prisma 操作数据
提供 restfulapi
